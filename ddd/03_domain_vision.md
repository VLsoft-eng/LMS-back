# Domain Vision

## Для кого LMS

| Роль | Кто это | Ключевая задача |
|------|---------|----------------|
| OWNER | Преподаватель-организатор курса | Создать класс, пригласить участников, настроить задания |
| TEACHER | Ассистент / второй преподаватель | Оценивать работы, управлять командами |
| STUDENT | Студент | Сдавать задания, получать и видеть оценки |
| Аноним | Новый пользователь | Зарегистрироваться, войти по инвайт-коду |

## Ключевые ценностные сценарии

1. **Учебный цикл**: Преподаватель создаёт задание -> студент сдаёт работу -> преподаватель оценивает -> студент видит результат.
2. **Командное обучение**: Преподаватель делит класс на команды -> команда сдаёт совместно -> преподаватель оценивает команду -> индивидуальные корректировки.
3. **Рубрикованное оценивание**: Преподаватель создаёт шаблон критериев -> прикрепляет к заданию -> оценивает по каждому критерию -> система автоматически вычисляет итог.
4. **Быстрая оценка (Quick Assignment)**: Упрощённый цикл для аудиторных работ без отслеживания дедлайна.

## Ключевые бизнес-метрики

| Метрика | Источник в коде |
|---------|----------------|
| Средний балл по классу | `ClassStatsDto.averageGrade`, нормализован 0-100 |
| Процент сданных работ | `ClassStatsDto` (submissions / assignments) |
| Просроченные дедлайны | `ClassStatsDto.missedDeadlines` per student |
