# Subdomain Map

| Subdomain | Тип | Обоснование |
|-----------|-----|-------------|
| Identity & Access | Generic | Стандартная регистрация/авторизация по JWT. Конкурентного преимущества не даёт. Можно заменить Keycloak/Auth0. |
| Class & Enrollment Management | Supporting | Организационный каркас. Важен для работы продукта, но не создаёт уникальной ценности сам по себе. |
| Assignment & Content | Core | Определяет учебный процесс: типы заданий, дедлайны, файловые вложения. Прямо влияет на ценность для пользователя. |
| Team Management | Supporting | Обеспечивает командное обучение. Важная, но вторичная функция. |
| **Grading & Assessment** | **Core** | Наиболее сложная бизнес-логика. Рубрики + критерии + бонусные мультипликаторы — уникальный дифференциатор. Сюда входит система критериев оценивания. |
| File Storage | Generic | Commodity S3-like хранилище. Бизнес-логики не содержит. |
| Analytics & Reporting | Supporting | Производная от Core; помогает принимать решения, но не создаёт ценность самостоятельно. |

**Система критериев оценивания** относится к Core subdomain **Grading & Assessment**. Именно здесь сосредоточена уникальная бизнес-логика — три типа критериев, формула расчёта PRIMARY + BONUS, иммутабельный снимок рубрики, нормализация — которая отличает LMS от простых систем выставления баллов.
